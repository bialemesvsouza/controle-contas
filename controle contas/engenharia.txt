Analisando seus dados, vejo que a estrutura precisa de alguns ajustes. Aqui está uma proposta melhorada:

## TABELA DESPESAS (Transação)
```sql
CREATE TABLE transacao (
    id_transacao INT PRIMARY KEY AUTO_INCREMENT,
    id_usuario INT,
    id_tipo INT,
    id_forma_pagamento INT,
    id_banco INT,
    descricao VARCHAR(255),
    valor_total DECIMAL(10,2),
    numero_parcelas INT,
    data_insercao DATETIME DEFAULT CURRENT_TIMESTAMP,
    tipo_transacao ENUM('despesa', 'receita'),
    detalhes TEXT,
    FOREIGN KEY (id_usuario) REFERENCES usuario(id),
    FOREIGN KEY (id_tipo) REFERENCES tipo(id),
    FOREIGN KEY (id_forma_pagamento) REFERENCES forma_pagamento(id),
    FOREIGN KEY (id_banco) REFERENCES banco(id)
);
```

## TABELA PARCELAS
```sql
CREATE TABLE parcela (
    id_parcela INT PRIMARY KEY AUTO_INCREMENT,
    id_transacao INT,
    id_usuario INT,
    numero_parcela INT,
    valor_parcela DECIMAL(10,2),
    data_vencimento DATE,
    data_pagamento DATE NULL,
    status ENUM('a_pagar', 'pago', 'atrasado'),
    forma_pagamento VARCHAR(100),
    banco_pagamento VARCHAR(100),
    baixado BOOLEAN DEFAULT FALSE,
    justificativa_baixa TEXT,
    FOREIGN KEY (id_transacao) REFERENCES transacao(id_transacao),
    FOREIGN KEY (id_usuario) REFERENCES usuario(id)
);
```

## TABELAS AUXILIARES
```sql
CREATE TABLE tipo (
    id_tipo INT PRIMARY KEY AUTO_INCREMENT,
    id_usuario INT,
    nome VARCHAR(100),
    categoria ENUM('despesa', 'receita'),
    cor VARCHAR(7),
    FOREIGN KEY (id_usuario) REFERENCES usuario(id)
);

CREATE TABLE forma_pagamento (
    id_forma_pagamento INT PRIMARY KEY AUTO_INCREMENT,
    nome VARCHAR(100) -- (Cartão, Dinheiro, PIX, etc.)
);

CREATE TABLE banco (
    id_banco INT PRIMARY KEY AUTO_INCREMENT,
    nome VARCHAR(100),
    id_usuario INT,
    FOREIGN KEY (id_usuario) REFERENCES usuario(id)
);
```

## CORREÇÕES IMPORTANTES:

1. **Separação correta**: A tabela principal é `transacao` (com tipo 'despesa' ou 'receita') e as parcelas ficam na tabela `parcela`

2. **Evitar redundância**: Não repetir `forma_pagamento` e `banco` em ambas tabelas

3. **Chaves estrangeiras**: Manter consistência nos nomes (ex: `id_transacao` em vez de `iddes`)

4. **Estrutura normalizada**: Informações de pagamento podem ficar na parcela se forem específicas por parcela
